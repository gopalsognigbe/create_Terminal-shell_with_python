### Analyse du fichier `Terminal.py` :

**Rôle** :  
Le fichier implémente un terminal interactif minimaliste permettant de naviguer dans le système de fichiers et d'exécuter des commandes système.

**Logique principale** :  
1. **Boucle REPL** : Le terminal affiche en permanence le répertoire courant comme invite de commande.
2. **Commandes** :
   - `cd` : Géré en interne pour changer de répertoire.
   - Autres commandes : Déléguées au shell système pour exécution.

**Fonctions importantes** :  
- `change_directory(path)` : Valide et change le répertoire courant.
- `execute_command(cmd)` : Exécute une commande système via `subprocess.run()`.

**Exemple d'utilisation** :  
```bash
/user/docs > pwd
/user/docs > cd /tmp
/tmp > 
```

**Problème critique** :  
L'utilisation de `shell=True` dans `subprocess.run()` expose le terminal à des injections de commandes. La correction recommandée est d'utiliser `shell=False` avec `shlex.split(cmd)` pour sécuriser l'exécution des commandes.

```python
subprocess.run(shlex.split(cmd), shell=False)
```

### Points clés à retenir :
- Ce fichier est un terminal interactif pour naviguer et exécuter des commandes.
- Il est vulnérable aux injections de commandes en raison de `shell=True`.
- Une correction simple consiste à utiliser `shell=False` avec `shlex.split(cmd)`.
