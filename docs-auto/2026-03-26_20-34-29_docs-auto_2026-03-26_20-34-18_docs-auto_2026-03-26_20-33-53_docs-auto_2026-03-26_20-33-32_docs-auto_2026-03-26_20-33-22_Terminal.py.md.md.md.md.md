### Analyse synthétique du fichier `Terminal.py` :

---

#### **Rôle principal**  
Interface shell minimaliste pour exécuter des commandes système avec gestion du répertoire courant.

---

#### **Cœur fonctionnel**  
1. **Boucle REPL** :  
   - Lit des commandes jusqu'à `exit`.  
   - Gère `cd` en interne, délègue le reste au système via `subprocess`.  

2. **Sécurité** :  
   - Risque d'injection via `shell=True` (à remplacer par `shell=False` + listes d'arguments).  

---

#### **Fonctions clés**  
- **`change_directory()`** : Navigation sécurisée avec vérification des chemins.  
- **`execute_command()`** : Exécution générique avec capture de sortie/erreurs.  
- **Prompt dynamique** : Affiche le répertoire courant.  

---

#### **Exemple type**  
```bash
~/projet $ python Terminal.py
~/projet $ ls -l
~/projet $ cd src
~/projet/src $ exit
```

---

#### **À retenir**  
- Pour usage interne uniquement (risques sécurité).  
- Architecture extensible pour ajouter des commandes maison.
