Analyse ultra-concise du fichier `Terminal.py` :

**Rôle** : Terminal interactif basique avec navigation système et exécution de commandes.

**Cœur logique** :
1. REPL simple avec prompt affichant le répertoire courant
2. Deux types de commandes :
   - `cd` (interne)
   - Autres (passées au shell)

**Fonctions clés** :
- `change_directory()` : gestion des chemins
- `execute_command()` : exécution non sécurisée via `subprocess.run(shell=True)`

**Exemple** :
```bash
/home > ls
/home > cd /tmp
```

**Alerte sécurité** :  
Vulnérabilité majeure d'injection de commandes due à `shell=True`. Solution minimale :
```python
subprocess.run(command.split(), shell=False)
```
