### Analyse ultra-concise du fichier `Terminal.py` :

**Rôle** :  
Shell minimaliste pour navigation système et exécution de commandes.

**Cœur fonctionnel** :  
- Boucle REPL (`read-eval-print-loop`) avec sortie sur `exit`  
- Routing basique :  
  - `cd` → gestion spéciale  
  - Autres commandes → exécution brute via `subprocess` (⚠️ sécurité)

**Fonctions clés** :  
1. `change_directory()` : wrapper sécurisé pour `os.chdir()`  
2. `execute_command()` : pipeline d'exécution avec gestion d'erreurs

**Cas type** :  
```bash
$ pwd  
/home/projet  
$ cd /tmp && ls | head -3  
$ exit  
```

**Note critique** :  
L'implémentation actuelle avec `shell=True` est dangereuse pour toute utilisation réelle (injection de commandes possible). Préférer `subprocess.run([cmd, arg1, arg2])`.
