### Analyse synthétique du fichier `Terminal.py` :

**Rôle principal**  
Shell minimaliste offrant :  
- Navigation système de fichiers  
- Exécution de commandes système  
- Support basique de pipelines  

**Logique clé**  
Boucle REPL avec :  
1. Prompt dynamique (basé sur le répertoire)  
2. Commandes internes (`cd`, `exit`)  
3. Délégation système via `subprocess`  

**Fonctions critiques**  
1. **`change_directory(path)`**  
   - Gère `cd` avec vérification du chemin  
   - Logique d'erreur simpliste  

2. **`execute_command(command)`**  
   - Exécution système via `subprocess.run(shell=True)`  
   - Support pipelines natif (risque de sécurité)  

**Cas d'usage typique**  
```bash
$ cd /tmp && ls -l | grep .log  # Navigation + filtrage  
$ exit  # Quitter  
```

**Alerte sécurité**  
- `shell=True` non sécurisé (injection possible)  
- Solution : Préférer `subprocess.run(args_list)` + parser dédié  

**Verdict**  
Proof of concept fonctionnel mais non sécurisé pour un usage réel. Améliorations nécessaires.
