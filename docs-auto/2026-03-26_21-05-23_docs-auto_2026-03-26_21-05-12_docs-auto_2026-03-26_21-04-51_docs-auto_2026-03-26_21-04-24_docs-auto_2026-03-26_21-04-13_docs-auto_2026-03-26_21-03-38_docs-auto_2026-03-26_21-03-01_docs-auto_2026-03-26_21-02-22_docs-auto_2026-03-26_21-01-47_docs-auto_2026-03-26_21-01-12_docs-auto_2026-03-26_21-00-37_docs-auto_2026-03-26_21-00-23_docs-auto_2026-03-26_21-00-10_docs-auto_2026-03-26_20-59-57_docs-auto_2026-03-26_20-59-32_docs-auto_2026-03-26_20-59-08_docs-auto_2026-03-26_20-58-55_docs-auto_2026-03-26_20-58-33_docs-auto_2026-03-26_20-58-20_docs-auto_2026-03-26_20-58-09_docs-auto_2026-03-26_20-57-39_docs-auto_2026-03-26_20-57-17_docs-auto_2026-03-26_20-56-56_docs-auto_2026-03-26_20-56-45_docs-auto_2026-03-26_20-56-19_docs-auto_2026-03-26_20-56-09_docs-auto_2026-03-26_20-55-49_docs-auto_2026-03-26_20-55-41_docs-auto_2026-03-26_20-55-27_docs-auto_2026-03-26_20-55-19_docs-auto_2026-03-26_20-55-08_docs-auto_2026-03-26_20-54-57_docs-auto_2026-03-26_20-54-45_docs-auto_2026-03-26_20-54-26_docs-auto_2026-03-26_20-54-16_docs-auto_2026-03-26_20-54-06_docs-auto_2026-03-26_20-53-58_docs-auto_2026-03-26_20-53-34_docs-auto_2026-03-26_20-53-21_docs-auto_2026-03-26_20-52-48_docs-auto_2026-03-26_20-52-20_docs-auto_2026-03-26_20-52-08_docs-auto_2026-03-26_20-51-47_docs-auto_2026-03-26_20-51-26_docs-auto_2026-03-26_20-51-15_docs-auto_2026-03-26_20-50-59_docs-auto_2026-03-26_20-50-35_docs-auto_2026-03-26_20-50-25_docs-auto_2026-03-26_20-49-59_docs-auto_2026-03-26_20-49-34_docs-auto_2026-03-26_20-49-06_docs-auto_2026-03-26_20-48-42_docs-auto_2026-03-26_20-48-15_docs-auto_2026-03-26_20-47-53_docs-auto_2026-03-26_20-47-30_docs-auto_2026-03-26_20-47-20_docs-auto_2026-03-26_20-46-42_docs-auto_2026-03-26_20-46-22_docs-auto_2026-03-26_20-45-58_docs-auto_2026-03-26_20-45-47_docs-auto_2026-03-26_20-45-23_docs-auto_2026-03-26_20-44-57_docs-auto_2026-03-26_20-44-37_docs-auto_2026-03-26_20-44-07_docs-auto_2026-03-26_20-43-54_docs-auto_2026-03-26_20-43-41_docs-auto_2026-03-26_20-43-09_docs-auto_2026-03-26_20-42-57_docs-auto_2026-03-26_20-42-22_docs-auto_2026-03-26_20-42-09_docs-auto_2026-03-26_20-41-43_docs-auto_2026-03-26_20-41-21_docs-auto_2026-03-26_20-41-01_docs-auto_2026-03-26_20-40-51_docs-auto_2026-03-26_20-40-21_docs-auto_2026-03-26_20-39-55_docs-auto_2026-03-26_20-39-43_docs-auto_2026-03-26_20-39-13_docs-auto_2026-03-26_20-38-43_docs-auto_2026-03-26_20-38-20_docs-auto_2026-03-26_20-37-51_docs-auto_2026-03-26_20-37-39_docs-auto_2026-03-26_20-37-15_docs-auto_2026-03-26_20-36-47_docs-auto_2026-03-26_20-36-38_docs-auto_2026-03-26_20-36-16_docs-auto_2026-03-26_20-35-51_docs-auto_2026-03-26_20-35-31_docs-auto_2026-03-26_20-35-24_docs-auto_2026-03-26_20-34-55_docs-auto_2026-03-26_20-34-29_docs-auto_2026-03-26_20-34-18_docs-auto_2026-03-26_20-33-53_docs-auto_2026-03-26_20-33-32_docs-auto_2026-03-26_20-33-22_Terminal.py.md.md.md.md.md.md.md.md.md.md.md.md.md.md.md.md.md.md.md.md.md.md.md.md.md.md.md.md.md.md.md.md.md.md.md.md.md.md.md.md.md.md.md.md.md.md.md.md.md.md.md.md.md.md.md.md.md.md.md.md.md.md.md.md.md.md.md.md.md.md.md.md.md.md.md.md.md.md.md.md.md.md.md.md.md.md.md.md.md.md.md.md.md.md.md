### **Analyse synthétique du fichier `Terminal.py`**

---

#### **Rôle**  
Terminal interactif basique avec :  
- Navigation système (`cd`, `pwd`)  
- Exécution de commandes shell (`ls`, etc.)  

---

#### **Logique principale**  
1. **REPL** : Boucle infinie `input → exécution → affichage`  
2. **État** : Affiche/maintient le répertoire courant  

---

#### **Fonctions clés**  
1. `change_directory(path)` :  
   - Change de dossier avec vérification du chemin  
2. `execute_command(cmd)` :  
   - Exécute des commandes via `subprocess` (risque d'injection)  

---

#### **Exemple**  
```bash
~/projet$ cd src  
~/projet/src$ ls -l
```

---

#### **Alerte**  
⚠ **Risque sécurité** :  
- `shell=True` permet des injections (ex: `; rm -rf /`)  

---

#### **Améliorations suggérées**  
1. Sécuriser `execute_command` (liste d'arguments au lieu de `shell=True`)  
2. Ajouter historique/autocomplétion  
3. Limiter les permissions  

*Se concentre sur l'essentiel, ignore les détails secondaires.*
