### Analyse synthétique de `Terminal.py`  

#### **Rôle principal**  
Le fichier `Terminal.py` est un prototype de shell minimaliste qui offre trois fonctionnalités principales :  
1. **Navigation système** : Possibilité de changer de répertoire avec la commande `cd`.  
2. **Exécution de commandes externes** : Capacité à exécuter des commandes système externes.  
3. **Support des pipelines** : Prise en charge basique des pipelines avec l'opérateur `|`.  

#### **Logique principale**  
Le cœur du système repose sur une boucle REPL (Read-Eval-Print Loop) qui :  
1. Affiche le répertoire courant dans l'invite de commande.  
2. Dispatch les commandes entre :  
   - **Commandes internes** : Gérées directement par le shell, comme `exit`.  
   - **Commandes externes** : Passées au shell système via `subprocess.run(shell=True)`.  

#### **Fonctions importantes**  
1. `change_directory(path)`  
   - Gère les chemins relatifs et absolus.  
   - Bloque les accès non valides pour assurer la sécurité.  

2. `execute_command(command)`  
   - Point sensible : utilise `shell=True`, ce qui peut introduire des risques d'injection de commandes.  

#### **Exemple d'utilisation**  
```bash 
$ cd /var/log && cat syslog | grep "error"  # Pipeline basique
```  

#### **Problèmes de sécurité**  
- **Vulnérabilité majeure** : L'utilisation de `shell=True` permet des injections de commandes, comme `; rm -rf /`.  
- **Solutions recommandées** :  
  - Utiliser `subprocess.run(["commande"])` pour éviter `shell=True`.  
  - Parser manuellement les pipelines.  
  - Sanitizer les entrées utilisateur pour prévenir les injections.  

#### **Conclusion**  
`Terminal.py` est une preuve de concept intéressante mais **potentiellement dangereuse en production** sans mesures de sécurité supplémentaires. Il est essentiel de renforcer les protections pour éviter les vulnérabilités critiques.
