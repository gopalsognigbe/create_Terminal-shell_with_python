### Analyse ultra-concise du fichier `Terminal.py` :

#### **Rôle**  
Implémente un terminal interactif basique pour exécuter des commandes système et naviguer dans les répertoires.

#### **Logique principale**  
1. Boucle REPL infinie affichant le répertoire courant (`cwd`) comme invite  
2. Deux types de commandes :  
   - **Internes** (ex: `cd`) gérées directement en Python  
   - **Externes** (ex: `ls`, `grep`) exécutées via `subprocess.run()` avec `shell=True` *(danger!)*  

#### **Fonctions clés**  
- `change_directory(path)` : Gère la navigation entre répertoires  
- `execute_command(command)` : Exécute les commandes shell via subprocess  

#### **Exemple type**  
```bash
~/projects > cd /var/log  
/var/log > grep error syslog  # Passe la commande au shell système
```

#### **Alerte sécurité**  
⚠️ **Dangereux en production** :  
- `shell=True` permet l'injection de commandes  
- Aucune validation/sanitization des entrées  
- Usage réservé à l'apprentissage en environnement contrôlé
