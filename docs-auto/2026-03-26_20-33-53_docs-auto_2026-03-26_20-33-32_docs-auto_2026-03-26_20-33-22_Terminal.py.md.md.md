### Analyse synthétique du fichier `Terminal.py` :

#### Rôle principal :
Le fichier `Terminal.py` sert d'interface shell simplifiée permettant d'exécuter des commandes système tout en gérant le répertoire courant de manière basique.

#### Logique principale :
1. **Boucle REPL** : Une boucle persistante (Read-Eval-Print Loop) qui attend et traite les commandes jusqu'à ce que l'utilisateur saisisse `exit`.
2. **Traitement spécialisé** : La commande `cd` est traitée de manière spécifique pour modifier le répertoire courant.
3. **Délégation générique** : Les autres commandes sont déléguées au système via le module `subprocess`.

#### Fonctions importantes :
1. **`change_directory()`** : Gère la navigation entre répertoires et prend en charge les erreurs de chemin.
2. **`execute_command()`** : Exécute les commandes système en capturant la sortie standard (`stdout`) et les erreurs (`stderr`).
3. **`display_prompt()`** : Affiche un prompt contextuel qui inclut le répertoire courant (CWD).

#### Exemple d'utilisation :
```bash
/projets > python Terminal.py
/projets > mkdir test
/projets > cd test
/projets/test > touch fichier.txt
/projets/test > exit
```

#### Notes de sécurité :
- **Attention** : L'utilisation de `shell=True` dans `subprocess` peut exposer à des injections de commandes.
- **Amélioration suggérée** : Utiliser `shell=False` avec une liste d'arguments pour les commandes complexes afin de renforcer la sécurité.
