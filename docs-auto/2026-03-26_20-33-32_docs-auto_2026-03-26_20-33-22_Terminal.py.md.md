### Analyse synthétique du fichier Terminal.py :

#### Rôle principal :
Interface shell simplifiée pour exécuter des commandes système avec gestion basique du répertoire courant.

#### Cœur logique :
1. **Boucle REPL** (Read-Eval-Print Loop) persistante jusqu'à la commande `exit`
2. **Traitement spécialisé** pour `cd` (modification d'état interne)
3. **Délégation générique** des autres commandes au système via subprocess

#### Fonctions critiques :
1. `change_directory()` : Logique de navigation (gère erreurs de chemin)
2. `execute_command()` : Pont système avec capture stdout/stderr
3. `display_prompt()` : Feedback contextuel (intègre le CWD)

#### Cas d'usage typique :
```bash
/projets > python Terminal.py
/projets > mkdir test
/projets > cd test
/projets/test > touch fichier.txt
/projets/test > exit
```

#### Notes de sécurité :
- **Attention** : L'usage de `shell=True` dans subprocess expose à des injections de commandes
- **Amélioration suggérée** : Utiliser `shell=False` avec liste d'arguments pour les commandes complexes
