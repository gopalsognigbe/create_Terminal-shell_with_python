Analyse ultra-concise du fichier `Terminal.py` :

**Rôle** :  
Implémente un terminal interactif minimaliste pour naviguer dans le système de fichiers et exécuter des commandes système.

**Logique principale** :  
1. Boucle REPL affichant le répertoire courant comme prompt
2. Deux types de commandes :
   - `cd` (géré en interne)
   - Autres commandes (déléguées au shell système)

**Fonctions clés** :  
- `change_directory(path)` : Validation et changement de répertoire
- `execute_command(cmd)` : Exécution shell via subprocess.run()

**Exemple type** :  
```bash
/user/docs > pwd
/user/docs > cd /tmp
/tmp > 
```

**Problème critique** :  
L'implémentation actuelle avec `shell=True` permet des injections de commandes. Correction minimale :
```python
subprocess.run(shlex.split(cmd), shell=False)
```
