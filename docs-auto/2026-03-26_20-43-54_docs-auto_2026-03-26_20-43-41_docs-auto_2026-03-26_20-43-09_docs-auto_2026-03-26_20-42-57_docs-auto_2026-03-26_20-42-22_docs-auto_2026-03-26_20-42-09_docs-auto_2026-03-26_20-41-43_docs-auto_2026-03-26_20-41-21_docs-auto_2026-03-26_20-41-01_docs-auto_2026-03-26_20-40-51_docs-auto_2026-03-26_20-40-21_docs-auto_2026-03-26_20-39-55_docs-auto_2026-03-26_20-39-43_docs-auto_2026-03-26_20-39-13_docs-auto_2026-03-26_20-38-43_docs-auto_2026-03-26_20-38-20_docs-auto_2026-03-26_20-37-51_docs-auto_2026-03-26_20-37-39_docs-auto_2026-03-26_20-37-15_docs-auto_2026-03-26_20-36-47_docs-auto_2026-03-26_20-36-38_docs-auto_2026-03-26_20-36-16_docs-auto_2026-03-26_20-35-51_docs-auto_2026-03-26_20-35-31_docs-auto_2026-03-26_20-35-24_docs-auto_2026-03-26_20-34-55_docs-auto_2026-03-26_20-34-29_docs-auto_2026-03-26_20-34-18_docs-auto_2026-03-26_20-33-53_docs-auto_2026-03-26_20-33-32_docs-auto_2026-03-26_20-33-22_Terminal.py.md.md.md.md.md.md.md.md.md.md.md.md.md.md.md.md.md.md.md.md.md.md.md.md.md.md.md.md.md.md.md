### Analyse synthétique de `Terminal.py`  

#### **Rôle du fichier**  
Prototype de shell minimaliste permettant :  
- Navigation dans le système de fichiers (`cd`).  
- Exécution de commandes externes.  
- Support basique des pipelines (ex: `|`).  

#### **Logique principale**  
Boucle REPL (Read-Eval-Print Loop) qui :  
1. Affiche un prompt dynamique avec le répertoire courant.  
2. Route les commandes :  
   - **Internes** (ex: `cd`, `exit`) → gérées par le script.  
   - **Externes** → déléguées au shell système via `subprocess.run(shell=True)`.  

#### **Fonctions clés**  
1. **`change_directory(path)`**  
   - Gère la navigation (chemins relatifs/absolus).  
   - Bloque les accès invalides.  

2. **`execute_command(command)`**  
   - Exécute des commandes externes via le shell.  
   - **Problème critique** : `shell=True` expose à des injections de commandes.  

#### **Exemple d'utilisation**  
```bash
$ cd /tmp && ls | grep ".log"  # Navigue vers `/tmp` et filtre les logs.  
```

#### **Alerte sécurité**  
- **Risque** : Injection via `shell=True` (ex: `; rm -rf /`).  
- **Correctifs recommandés** :  
  - Remplacer `shell=True` par des listes d'arguments (ex: `subprocess.run(["ls"])`).  
  - Parser manuellement les pipelines.  
  - Valider les entrées utilisateur.  

#### **Conclusion**  
Proof-of-concept fonctionnel mais **non sécurisé** pour un usage réel. Améliorations nécessaires en sécurité et gestion des commandes.
