### Analyse ultra-concise du fichier `Terminal.py` :

#### **Rôle**  
Fichier implémentant un terminal interactif simplifié permettant d'exécuter des commandes système et de naviguer dans le système de fichiers.

#### **Logique principale**  
1. Boucle REPL (Read-Eval-Print Loop) infinie affichant le répertoire courant (`cwd`)  
2. Distinction entre :  
   - Commandes internes (comme `cd`) gérées directement par Python  
   - Commandes externes (comme `ls`, `cat`) exécutées via `subprocess.run()` avec `shell=True` *(méthode risquée)*  

#### **Fonctions importantes**  
- `change_directory()` : Modifie le répertoire de travail courant  
- `execute_command()` : Délègue l'exécution des commandes externes au shell système  

#### **Exemple d'utilisation**  
```bash
/home/user > cd /tmp  
/tmp > ls -l  # Liste les fichiers via le shell système
```

#### **Avertissement critique**  
Conçu pour un usage éducatif uniquement. La méthode `shell=True` pose des risques majeurs d'injection de commandes. À proscrire en environnement réel.
