### Analyse synthétique du fichier `Terminal.py` :

#### **Rôle du fichier**  
Implémentation minimaliste d'un shell en Python avec des fonctionnalités basiques (navigation et exécution de commandes).

#### **Logique principale**  
1. Boucle REPL simple avec invite personnalisée (`cwd >`)
2. Distinction entre commandes internes (ex: `cd`) et externes (ex: `ls`)
3. Exécution via `subprocess.run` (avec risque de sécurité via `shell=True`)

#### **Fonctions clés**  
- `change_directory()` : Gère la navigation entre répertoires
- `execute_command()` : Lance les commandes système via subprocess

#### **Exemple type**  
```bash
/home/user > cd /tmp
/tmp > ls *.log
```

#### **Avertissements**  
1. **Danger** : `shell=True` permet l'injection de commandes (ex: `; rm -rf /`)
2. **Limitations** : Pas de gestion des pipes/complexités shell modernes
3. **Usage** : Uniquement éducatif (POC), non adapté à la production

**Valeur réelle** : Bon exercice pédagogique pour comprendre le fonctionnement d'un shell basique, mais nécessiterait des sécurisations (comme `shell=False` + parsing manuel) pour un usage réel.
