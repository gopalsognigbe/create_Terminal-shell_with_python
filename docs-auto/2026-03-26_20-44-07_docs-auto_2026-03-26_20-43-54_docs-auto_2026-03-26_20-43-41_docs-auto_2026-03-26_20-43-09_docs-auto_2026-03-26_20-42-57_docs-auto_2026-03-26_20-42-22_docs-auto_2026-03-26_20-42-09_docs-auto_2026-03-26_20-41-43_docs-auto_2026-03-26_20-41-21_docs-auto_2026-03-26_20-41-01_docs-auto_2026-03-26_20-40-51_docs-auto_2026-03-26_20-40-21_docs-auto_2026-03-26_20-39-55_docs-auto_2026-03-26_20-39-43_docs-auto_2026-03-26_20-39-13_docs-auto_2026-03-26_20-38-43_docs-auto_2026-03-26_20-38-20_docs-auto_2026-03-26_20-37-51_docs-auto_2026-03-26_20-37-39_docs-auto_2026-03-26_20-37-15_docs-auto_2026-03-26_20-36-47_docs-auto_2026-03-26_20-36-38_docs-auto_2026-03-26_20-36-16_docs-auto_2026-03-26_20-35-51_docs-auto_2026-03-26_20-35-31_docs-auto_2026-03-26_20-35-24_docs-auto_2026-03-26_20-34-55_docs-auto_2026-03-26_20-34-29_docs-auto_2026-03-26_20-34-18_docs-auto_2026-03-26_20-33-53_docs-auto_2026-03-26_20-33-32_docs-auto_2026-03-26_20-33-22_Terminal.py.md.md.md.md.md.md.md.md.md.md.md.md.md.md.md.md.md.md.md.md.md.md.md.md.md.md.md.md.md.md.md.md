### Analyse synthétique de `Terminal.py`  

#### **Rôle principal**  
Prototype de shell minimaliste avec 3 fonctionnalités clés :  
1. Navigation système (`cd`)  
2. Exécution de commandes externes  
3. Support basique des pipelines (`|`)  

#### **Cœur du système**  
Boucle REPL simple qui :  
1. Affiche le répertoire courant dans le prompt  
2. Dispatch les commandes :  
   - **Internes** : gérées directement (ex: `exit`)  
   - **Externes** : passées au shell système via `subprocess.run(shell=True)`  

#### **Fonctions critiques**  
1. `change_directory(path)`  
   - Gère les chemins (relatifs/absolus)  
   - Bloque les accès non valides  

2. `execute_command(command)`  
   - Point sensible : utilise `shell=True` (risque d'injection)  

#### **Exemple type**  
```bash 
$ cd /var/log && cat syslog | grep "error"  # Pipeline basique
```

#### **Alerte majeure**  
- **Vulnérabilité** : `shell=True` permet des injections (ex: `; rm -rf /`)  
- **Solutions** :  
  - Préférer `subprocess.run(["ls"])`  
  - Parser les pipelines manuellement  
  - Sanitizer les entrées  

#### **Verdict**  
POC intéressant mais **dangereux en production** sans renforcement de la sécurité.
