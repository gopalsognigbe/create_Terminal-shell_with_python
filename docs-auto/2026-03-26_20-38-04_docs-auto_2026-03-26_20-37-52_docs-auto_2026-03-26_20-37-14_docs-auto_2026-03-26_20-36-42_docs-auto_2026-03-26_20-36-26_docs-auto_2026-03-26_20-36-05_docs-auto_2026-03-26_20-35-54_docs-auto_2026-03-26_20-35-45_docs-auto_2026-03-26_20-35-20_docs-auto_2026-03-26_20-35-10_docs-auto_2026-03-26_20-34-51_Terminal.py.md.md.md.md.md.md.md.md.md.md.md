### Analyse synthétique du fichier `Terminal.py` :

---

#### **Rôle principal**  
Implémente un terminal interactif minimaliste avec :
- Gestion native des commandes `cd` (changement de répertoire)  
- Commande `exit` pour quitter  
- Exécution générique des autres commandes système  

---

#### **Logique cœur**  
1. **REPL (Read-Eval-Print Loop)** :  
   - Lit une commande → Exécute → Affiche le résultat  
   - Boucle jusqu'à `exit`  
2. **Traitement spécialisé** :  
   - `cd` : Utilise `os.chdir()`  
   - Autres commandes : Déléguées à `subprocess.run()`  

---

#### **Fonctions critiques**  
1. **`os.chdir()`** :  
   - Change le répertoire de travail (commande `cd`)  
   - Inclut une vérification d'erreur  
2. **`subprocess.run(shell=True)`** :  
   - Exécute les commandes shell (risque de sécurité)  

---

#### **Exemple type**  
```bash
/home/user $ cd /tmp      # Changement de répertoire  
/tmp $ ls -l              # Liste les fichiers  
/tmp $ exit               # Ferme le terminal  
```

---

#### **Alerte sécurité**  
- **`shell=True`** :  
  - Permet l'injection de commandes (ex: `; rm -rf /`)  
  - **Solution** : Préférer `shell=False` + liste d'arguments  

---

### En résumé :  
- **Pour** : Terminal basique avec `cd`/`exit`  
- **Problème majeur** : Vulnérabilité via `shell=True`  
- **Usage** : Navigation et commandes système simples
