### Analyse concise du fichier `Terminal.py` :

#### **Rôle**  
Le fichier `Terminal.py` implémente un shell minimaliste qui offre une interface REPL (Read-Eval-Print Loop) pour exécuter des commandes système, incluant la navigation entre répertoires et des pipelines basiques.

#### **Logique centrale**  
Le programme repose sur une boucle REPL simple :  
1. **Affiche l'invite** : Montre le répertoire courant (`pwd`).  
2. **Traite la commande** : Identifie si la commande est interne (comme `cd`) ou externe (comme `ls`).  
3. **Renvoie le résultat** : Exécute la commande et affiche le résultat à l'utilisateur.

#### **Fonctions clés**  
1. **`change_directory(path)`**  
   - Permet de naviguer entre répertoires en vérifiant que le chemin existe.  
2. **`execute_command(command)`**  
   - Exécute des commandes externes via `subprocess.run(shell=True)`, ce qui introduit un risque de sécurité dû à l'injection de commandes.

#### **Exemple d'utilisation**  
```bash
$ cd /tmp && ls -l | grep ".log"  # Utilisation d'un pipeline basique
```

#### **Avertissements**  
- **Vulnérabilités** : L'utilisation de `shell=True` dans `subprocess.run` expose à des risques d'injection de commandes.  
- **Limitations** : Le shell est très basique et ne convient pas pour des utilisations avancées ou en production.  

En résumé : Il s'agit d'un POC (Proof of Concept) éducatif, non sécurisé pour une utilisation en production.
