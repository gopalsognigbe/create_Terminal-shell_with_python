### Analyse du fichier `Terminal.py` :

---

#### **Rôle du fichier** :
Le fichier `Terminal.py` agit comme une interface shell simplifiée, permettant d'exécuter des commandes système tout en gérant le répertoire courant.

---

#### **Logique principale** :
1. **Boucle REPL** : Une boucle persistante (Read-Eval-Print Loop) qui attend des commandes utilisateur jusqu'à ce que `exit` soit saisi.
2. **Traitement spécialisé** : La commande `cd` est gérée spécifiquement pour changer le répertoire courant.
3. **Délégation générique** : Les autres commandes sont exécutées via `subprocess`, en capturant la sortie standard et les erreurs.

---

#### **Fonctions importantes** :
1. **`change_directory()`** : Gère la navigation entre répertoires et vérifie les chemins valides.
2. **`execute_command()`** : Exécute les commandes système, en encapsulant `subprocess` pour capturer `stdout` et `stderr`.
3. **`display_prompt()`** : Affiche un prompt contextuel incluant le répertoire courant (CWD).

---

#### **Exemple d'utilisation** :
```bash
/projets > python Terminal.py
/projets > mkdir test
/projets > cd test
/projets/test > touch fichier.txt
/projets/test > exit
```

---

#### **Points clés à retenir** :
- **Attention** : L'utilisation de `shell=True` dans `subprocess` peut exposer à des injections de commandes.
- **Amélioration suggérée** : Préférer `shell=False` avec une liste d'arguments pour les commandes complexes afin de renforcer la sécurité.
