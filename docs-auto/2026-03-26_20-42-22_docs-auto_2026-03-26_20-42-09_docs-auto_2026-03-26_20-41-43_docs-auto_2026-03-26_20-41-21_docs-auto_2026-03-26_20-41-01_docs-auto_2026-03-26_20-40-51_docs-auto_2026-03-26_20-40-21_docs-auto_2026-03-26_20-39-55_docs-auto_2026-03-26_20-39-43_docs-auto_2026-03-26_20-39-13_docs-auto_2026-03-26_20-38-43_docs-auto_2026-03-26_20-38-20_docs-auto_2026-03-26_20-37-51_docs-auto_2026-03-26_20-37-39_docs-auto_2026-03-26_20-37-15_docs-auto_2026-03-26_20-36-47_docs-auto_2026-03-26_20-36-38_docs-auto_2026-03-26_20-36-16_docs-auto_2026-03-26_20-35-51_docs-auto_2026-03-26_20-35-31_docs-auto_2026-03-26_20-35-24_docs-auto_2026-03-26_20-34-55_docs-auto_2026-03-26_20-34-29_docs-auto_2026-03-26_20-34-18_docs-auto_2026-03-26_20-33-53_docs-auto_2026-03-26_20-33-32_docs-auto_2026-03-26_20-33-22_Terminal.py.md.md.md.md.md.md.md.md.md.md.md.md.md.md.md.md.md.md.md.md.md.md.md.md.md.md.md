### **Analyse ciblée du fichier `Terminal.py`**

#### **Rôle principal**  
Prototype de shell minimaliste avec :  
- Navigation système (`cd`)  
- Exécution de commandes externes  
- Support basique des pipelines  

#### **Cœur fonctionnel**  
1. **Boucle REPL** :  
   - Prompt dynamique (affiche le répertoire courant)  
   - Découpage et routage des commandes  

2. **Deux modes d'exécution** :  
   - Commandes internes (ex: `cd`, `exit`)  
   - Commandes externes (déléguées à `subprocess.run` avec `shell=True`)  

#### **Fonctions clés**  
- **`change_directory(path)`**  
  - Gère les chemins relatifs/absolus  
  - Bloque les chemins invalides  

- **`execute_command(command)`** (Critique)  
  - Exécution directe via le shell système  
  - Permet les pipelines mais vulnérable aux injections  

#### **Exemple type**  
```bash
$ cd ~/projects && ls *.py | wc -l  # Compte les fichiers Python  
```

#### **Alerte sécurité**  
⚠ **`shell=True`** :  
- Accepte tout input non filtré (ex: `; rm -rf /`)  
- Solution : Préférer `subprocess.run(['ls'])` et parser les pipes manuellement  

#### **En bref**  
Proof-of-concept utile mais dangereux en l'état. À réécrire avec :  
- Désactivation de `shell=True`  
- Parsing strict des commandes  
- Gestion dédiée des opérateurs (`|`, `&&`)
