### Analyse concise du fichier `Terminal.py` :

---

### **Rôle principal**  
Le fichier implémente un shell minimaliste permettant d'exécuter des commandes système tout en gérant le répertoire courant. Il fonctionne comme un REPL (Read-Eval-Print Loop) interactif en ligne de commande.

---

### **Logique principale**  
1. **Boucle REPL** :  
   - Lit les commandes saisies par l'utilisateur jusqu'à ce que la commande `exit` soit exécutée.  
   - Traite la commande `cd` pour changer de répertoire et délègue les autres commandes au système via `subprocess.run`.  
2. **Sécurité** :  
   - L'utilisation de `shell=True` dans `subprocess.run` expose le code à des risques d'injection de commandes. Pour sécuriser le code, il est recommandé d'utiliser `shell=False` et de passer les arguments sous forme de liste.

---

### **Fonctions importantes**  
1. **`change_directory()`** :  
   - Permet de naviguer entre les répertoires après avoir vérifié la validité du chemin.  
2. **`execute_command()`** :  
   - Exécute les commandes système et capture la sortie standard ainsi que les erreurs.  
3. **Prompt dynamique** :  
   - Affiche en temps réel le répertoire courant dans l'invite de commande pour améliorer l'expérience utilisateur.  

---

### **Exemple d'utilisation**  
```bash
~/projet $ python Terminal.py
~/projet $ ls -l
~/projet $ cd src
~/projet/src $ exit
```

---

### **Points clés** :  
- Ce fichier offre une architecture extensible pour ajouter des commandes personnalisées.  
- Il est conçu pour un usage interne en raison des vulnérabilités liées à `shell=True`.  
- Peut être amélioré en sécurisant les appels système et en ajoutant des fonctionnalités supplémentaires.
