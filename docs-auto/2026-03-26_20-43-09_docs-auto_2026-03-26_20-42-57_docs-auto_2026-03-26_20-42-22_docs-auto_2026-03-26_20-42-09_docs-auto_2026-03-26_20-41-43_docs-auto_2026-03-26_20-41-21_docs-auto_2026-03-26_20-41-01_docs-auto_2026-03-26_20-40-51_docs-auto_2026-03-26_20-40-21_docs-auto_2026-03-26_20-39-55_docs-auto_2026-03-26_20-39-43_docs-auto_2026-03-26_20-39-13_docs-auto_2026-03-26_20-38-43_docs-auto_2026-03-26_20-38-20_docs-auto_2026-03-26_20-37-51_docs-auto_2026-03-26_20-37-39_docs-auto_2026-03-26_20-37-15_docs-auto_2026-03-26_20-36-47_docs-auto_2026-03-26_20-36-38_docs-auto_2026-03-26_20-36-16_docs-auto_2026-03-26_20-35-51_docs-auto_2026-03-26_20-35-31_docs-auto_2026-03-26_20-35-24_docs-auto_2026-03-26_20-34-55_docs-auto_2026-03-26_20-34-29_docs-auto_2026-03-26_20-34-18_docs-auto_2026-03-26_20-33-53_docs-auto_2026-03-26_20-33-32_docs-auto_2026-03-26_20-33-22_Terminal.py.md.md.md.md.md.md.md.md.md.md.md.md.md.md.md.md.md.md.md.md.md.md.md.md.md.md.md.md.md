### **Analyse synthétique de `Terminal.py`**

#### **Rôle**  
Prototype de shell minimaliste avec :  
- Navigation système (`cd`)  
- Exécution de commandes externes  
- Support basique des pipelines  

#### **Logique clé**  
- Boucle REPL avec prompt dynamique (affiche le répertoire courant)  
- Routing des commandes :  
  - **Internes** (ex: `cd`, `exit`) → Gérées par le script  
  - **Externes** → Déléguées au shell via `subprocess.run(shell=True)`  

#### **Fonctions critiques**  
1. **`change_directory(path)`** :  
   - Gère les chemins (relatifs/absolus)  
   - Bloque les accès invalides  

2. **`execute_command(command)`** :  
   - Exécution directe via shell système  
   - Vulnérabilité : Injection de commandes possible (pas de filtrage)  

#### **Exemple type**  
```bash
$ cd /tmp && ls | grep ".log"  # Navigation + pipeline
```

#### **Alerte majeure**  
**`shell=True`** → Danger d'injection (ex: `; rm -rf /`).  
**Correctifs suggérés** :  
- Désactiver `shell=True`  
- Parser manuellement les commandes/pipes  
- Utiliser des listes d'arguments (`subprocess.run(['ls'])`)  

#### **Conclusion**  
Proof-of-concept utile mais **non sécurisé** en production. Nécessite un parsing strict pour être viable.
