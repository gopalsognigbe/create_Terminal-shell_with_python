### **Synthèse du fichier `Terminal.py`**

---

### **Rôle du fichier**  
Le fichier `Terminal.py` implémente un **terminal interactif simplifié** permettant :  
- La **navigation dans le système de fichiers** (changer de répertoire).  
- L'**exécution de commandes système basiques** (`ls`, `cd`, `pwd`, etc.).  

---

### **Logique principale**  
1. **Boucle REPL** (Read-Eval-Print Loop) :  
   - Lit les commandes entrées par l'utilisateur, les interprète, et exécute les actions correspondantes.  
   - Affiche les résultats ou les erreurs après chaque commande.  

2. **Gestion des chemins** :  
   - Conserve l'état du répertoire courant (chemin actuel).  
   - Met à jour dynamiquement le prompt pour refléter le répertoire actuel.  

---

### **Fonctions importantes**  
1. **`change_directory(path)`** :  
   - Permet de naviguer vers un répertoire spécifié (relatif ou absolu).  
   - Vérifie que le répertoire existe avant de procéder.  

2. **`execute_command(cmd)`** :  
   - Exécute les commandes système via le module `subprocess`.  
   - Utilise `shell=True`, ce qui expose le terminal à des **injections de commandes malveillantes**.  

---

### **Exemple d'utilisation**  
Voici comment utiliser le terminal interactif :  
```bash
~/project$ cd src          # Navigation dans le répertoire "src"  
~/project/src$ ls -l       # Liste les fichiers dans le répertoire actuel  
~/project/src$ pwd         # Affiche le chemin absolu du répertoire actuel  
```

---

### **Points clés**  
1. **Vulnérabilité majeure** :  
   - L'utilisation de `shell=True` dans `execute_command` rend le terminal vulnérable à des **injections de commandes malveillantes**.  

2. **Évolutions possibles** :  
   - Ajouter un **historique des commandes**.  
   - Implémenter l'**auto-complétion** pour améliorer l'expérience utilisateur.  
   - Gérer les **permissions** pour sécuriser l'exécution des commandes.  

3. **Atout principal** :  
   - Architecture simple et fonctionnelle pour un terminal minimaliste.
