### Analyse du fichier `Terminal.py` :

#### **Rôle principal**  
Implémente un terminal interactif simplifié pour exécuter des commandes système et naviguer dans le système de fichiers.

#### **Logique clé**  
1. **REPL (Read-Eval-Print Loop)** :  
   - Affiche en permanence le répertoire courant (`cwd`) comme invite  
   - Lit les commandes utilisateur jusqu'à `exit`  

2. **Traitement des commandes** :  
   - **Commandes internes** (ex: `cd`) gérées en Python  
   - **Commandes externes** (ex: `ls`, `cat`) déléguées au shell via `subprocess.run()`  

#### **Fonctions critiques**  
1. `change_directory(path)` :  
   - Gère la navigation entre répertoires avec vérification d'existence  
   - Met à jour le `cwd`  

2. `execute_command(command)` :  
   - Exécute les commandes shell avec `subprocess.run(shell=True)`  
   - Affiche la sortie/erreur  

#### **Exemple type**  
```bash
/home/user > cd /tmp
/tmp > ls -l  # Lance un processus shell pour lister les fichiers
```

#### **Avertissement majeur**  
🔴 **Risque d'injection de commandes** :  
- L'usage de `shell=True` sans sanitisation permet l'exécution de commandes malveillantes  
- Absence totale de contrôle des entrées utilisateur  
- Usage déconseillé en dehors de contextes isolés/éducation
