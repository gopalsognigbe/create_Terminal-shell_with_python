### Analyse synthétique du fichier `Terminal.py` :

**Rôle** :  
Terminal interactif basique gérant les commandes système, avec traitement spécial pour `cd` (changement de répertoire) et `exit`.

**Logique clé** :  
1. Boucle infinie pour l'interactivité  
2. Détection et traitement personnalisé de `cd` (via `os.chdir()`) et `exit`  
3. Délégation des autres commandes à `subprocess.run(shell=True)`

**Fonctions critiques** :  
- `os.chdir()` pour la navigation  
- `subprocess.run()` (avec risque sécurité via `shell=True`)  
- Gestion d'erreurs basique (surtout pour `cd`)

**Exemple type** :  
```bash
/user $ cd /tmp  # Change directory  
/tmp $ ls        # Liste fichiers  
/tmp $ exit      # Quitte  
```

**Alerte majeure** :  
L'usage de `shell=True` est un risque de sécurité (injection de commandes) à corriger en priorité. Les autres remarques sont cosmétiques.
