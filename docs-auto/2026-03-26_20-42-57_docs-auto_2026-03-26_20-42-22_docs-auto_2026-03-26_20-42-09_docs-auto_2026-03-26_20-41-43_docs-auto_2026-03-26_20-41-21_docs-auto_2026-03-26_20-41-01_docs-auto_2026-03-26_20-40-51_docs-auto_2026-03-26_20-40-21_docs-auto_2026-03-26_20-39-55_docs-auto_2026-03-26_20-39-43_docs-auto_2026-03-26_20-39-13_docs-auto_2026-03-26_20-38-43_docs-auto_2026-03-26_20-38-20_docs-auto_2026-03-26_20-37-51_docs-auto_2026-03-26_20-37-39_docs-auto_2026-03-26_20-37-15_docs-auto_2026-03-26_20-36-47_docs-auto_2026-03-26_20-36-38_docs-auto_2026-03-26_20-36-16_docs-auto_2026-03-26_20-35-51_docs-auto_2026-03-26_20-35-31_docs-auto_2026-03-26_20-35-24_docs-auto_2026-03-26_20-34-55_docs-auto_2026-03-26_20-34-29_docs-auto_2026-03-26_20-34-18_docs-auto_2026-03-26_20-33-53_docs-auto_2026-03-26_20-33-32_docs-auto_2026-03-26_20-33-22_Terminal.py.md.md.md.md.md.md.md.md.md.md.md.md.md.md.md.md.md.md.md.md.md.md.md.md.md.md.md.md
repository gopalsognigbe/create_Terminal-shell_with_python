### **Analyse ciblée du fichier `Terminal.py`**

#### **Rôle principal**  
Le fichier `Terminal.py` est un prototype de shell minimaliste qui permet :  
- La navigation dans le système de fichiers (`cd`)  
- L'exécution de commandes externes  
- Le support basique des pipelines  

#### **Logique principale**  
Le fichier met en place une boucle REPL (Read-Eval-Print Loop) qui :  
- Affiche un prompt dynamique indiquant le répertoire courant  
- Découpe et route les commandes entrées par l'utilisateur  

Il distingue deux modes d'exécution :  
1. **Commandes internes** : comme `cd` ou `exit`, gérées directement par le script  
2. **Commandes externes** : déléguées au shell système via `subprocess.run` avec `shell=True`  

#### **Fonctions importantes**  
- **`change_directory(path)`** :  
  - Gère les chemins relatifs et absolus  
  - Bloque les chemins invalides  

- **`execute_command(command)`** :  
  - Exécute directement les commandes via le shell système  
  - Supporte les pipelines mais est vulnérable aux injections de commandes  

#### **Exemple d'utilisation**  
```bash
$ cd ~/projects && ls *.py | wc -l  # Compte les fichiers Python dans le répertoire "projects"  
```

#### **Alerte sécurité**  
⚠ **`shell=True`** :  
- Cette option permet l'exécution de toute entrée utilisateur non filtrée, ce qui introduit des risques d'injection de commandes (ex: `; rm -rf /`)  
- **Solution recommandée** :  
  - Désactiver `shell=True` et utiliser `subprocess.run(['commande'])`  
  - Parser manuellement les pipelines et les opérateurs (`|`, `&&`)  

#### **En bref**  
Le fichier est un proof-of-concept fonctionnel mais dangereux en l'état. Pour le rendre sécurisé, il faut :  
- Désactiver `shell=True`  
- Implémenter un parsing strict des commandes  
- Gérer explicitement les opérateurs de shell (`|`, `&&`, etc.)
