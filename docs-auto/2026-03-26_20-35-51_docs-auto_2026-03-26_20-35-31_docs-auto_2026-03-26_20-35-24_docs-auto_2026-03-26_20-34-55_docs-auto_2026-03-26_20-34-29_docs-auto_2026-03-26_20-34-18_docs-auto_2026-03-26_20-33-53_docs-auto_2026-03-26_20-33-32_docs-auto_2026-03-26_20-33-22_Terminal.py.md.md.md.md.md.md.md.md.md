### Analyse concise du fichier `Terminal.py` :

**Rôle du fichier** :  
Le fichier implémente un shell minimaliste interactif permettant de naviguer dans les répertoires et d'exécuter des commandes système.

**Logique principale** :  
1. **Boucle infinie** : Le terminal fonctionne en boucle continue, attendant les commandes de l'utilisateur jusqu'à ce que la commande `exit` soit saisie pour quitter.  
2. **Gestion des commandes** :  
   - `cd` est traité comme une commande spéciale pour changer de répertoire.  
   - Les autres commandes sont exécutées via `subprocess.run`, mais avec `shell=True`, ce qui pose un problème de sécurité.  

**Fonctions importantes** :  
- **`change_directory()`** : Gère le changement de répertoire de manière sécurisée.  
- **`execute_command()`** : Exécute les commandes système et gère les erreurs potentielles.  

**Exemple d'utilisation** :  
```bash
~/projet $ ls   # Liste les fichiers du répertoire courant  
~/projet $ cd src   # Change de répertoire  
~/projet/src $ exit   # Quitte le terminal  
```

**À retenir** :  
- Le fichier est extensible pour ajouter des commandes personnalisées.  
- La sécurité doit être améliorée en utilisant `shell=False` et en passant les commandes sous forme de liste d'arguments.
