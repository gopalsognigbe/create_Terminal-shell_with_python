### Analyse concise du fichier `Terminal.py` :

#### **Rôle du fichier**  
Ce fichier implémente un terminal interactif basique permettant d'exécuter des commandes système et de naviguer dans les répertoires.

#### **Logique principale**  
1. Une boucle REPL infinie affiche le répertoire courant (`cwd`) comme invite de commande.  
2. Les commandes sont séparées en deux catégories :  
   - **Internes** (ex: `cd`) : gérées directement en Python.  
   - **Externes** (ex: `ls`, `grep`) : exécutées via `subprocess.run()` avec `shell=True`.  

#### **Fonctions importantes**  
- `change_directory(path)` : Permet de naviguer entre les répertoires en modifiant le répertoire courant.  
- `execute_command(command)` : Exécute les commandes shell externes en utilisant `subprocess.run()`.  

#### **Exemple d'utilisation**  
```bash
~/projects > cd /var/log  
/var/log > grep error syslog  # Transmet la commande au shell système
```

#### **Alerte sécurité**  
⚠️ **Dangereux en production** :  
- `shell=True` expose à l'injection de commandes.  
- Aucune validation ou sanitisation des entrées utilisateur.  
- À éviter en production, réservé pour des environnements contrôlés ou pédagogiques.
