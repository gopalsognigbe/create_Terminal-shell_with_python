### Analyse du fichier `Terminal.py` :

**Rôle du fichier** :  
Le fichier implémente un terminal simplifié interactif permettant d'exécuter des commandes système, avec un traitement spécifique pour le changement de répertoire (`cd`) et la sortie (`exit`).

**Logique principale** :  
1. Boucle infinie qui attend des commandes utilisateur.  
2. Traitement spécial pour la commande `cd` en utilisant `os.chdir()`.  
3. Traitement spécial pour la commande `exit` pour quitter la boucle.  
4. Exécution des autres commandes via `subprocess.run()` avec `shell=True`.  

**Fonctions importantes** :  
- `os.chdir()` : Change le répertoire courant.  
- `subprocess.run()` : Exécute les commandes système.  
- Gestion d'erreur pour les échecs de `cd`.  

**Exemple d'utilisation** :  
```bash
/home/user > ls
file1.txt file2.txt
/home/user > cd Documents
/home/user/Documents > exit
```  
Le terminal affiche le répertoire courant avant chaque commande et exécute les commandes système normales.  

**Points à noter** :  
- Utilisation de `shell=True`, ce qui peut poser des risques de sécurité (injection de commandes).  
- Erreur de syntaxe : `stderrb` au lieu de `stderr`.  
- Message parasite : "ex jjitnb pour sortir" semblant être une erreur ou un texte inutile.
