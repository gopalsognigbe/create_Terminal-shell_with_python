### Analyse du fichier Terminal.py :

#### **Rôle du fichier**  
Ce fichier implémente un terminal interactif simplifié en Python qui :  
- Exécute des commandes système (comme dans un shell classique)  
- Gère le changement de répertoire (`cd`)  
- Permet de quitter le terminal (`exit`)  

---

#### **Logique principale**  
1. Boucle infinie (`while True`) attendant des commandes utilisateur.  
2. Traitement spécial pour `cd` (changement de répertoire) et `exit`.  
3. Pour les autres commandes, utilisation de `subprocess.run()` pour exécuter la commande système et afficher les résultats (sortie standard + erreurs).  

---

#### **Fonctions importantes**  
1. **`os.chdir()`** : Change le répertoire de travail (pour la commande `cd`).  
2. **`subprocess.run()`** :  
   - `shell=True` : Exécute la commande via le shell système.  
   - `capture_output=True` : Récupère la sortie de la commande.  
   - `universal_newlines=True` : Convertit les sorties en texte (au lieu d'octets).  

---

#### **Exemple d'utilisation**  
```bash
/home/user > ls  
# Affiche le contenu du répertoire (comme un terminal normal)  

/home/user > cd /tmp  
/tmp > pwd  
/tmp  

/tmp > exit  
# Quitte le terminal  
```

---

#### **Points clés à retenir**  
- **Simplicité** : Pas de gestion avancée des erreurs (ex: permissions) ou des commandes complexes (pipes, redirections).  
- **Sécurité** : `shell=True` peut être risqué si des entrées non filtrées sont utilisées (injection de commandes).  
- **Usage** : Utile pour un outil interne basique ou un prototype.  

Ignorer les détails comme les messages secondaires (`"exit pour sortir"`) ou la gestion minimale des erreurs.
