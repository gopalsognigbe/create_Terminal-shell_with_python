### Analyse du fichier Terminal.py :

#### Rôle du fichier :
Ce fichier implémente un terminal interactif simplifié qui permet d'exécuter des commandes système dans un environnement contrôlé. Il agit comme une interface entre l'utilisateur et le système d'exploitation.

#### Logique principale :
1. Boucle infinie qui attend des commandes utilisateur
2. Traitement spécial pour la commande `cd` (changement de répertoire)
3. Gestion de la commande `exit` pour quitter
4. Exécution générique des autres commandes via `subprocess.run()`

#### Fonctions importantes :
- `os.chdir()` : Change le répertoire de travail courant
- `subprocess.run()` : Exécute les commandes système et capture leur sortie
- `os.getcwd()` : Affiche le répertoire courant dans l'invite de commande

#### Exemple d'utilisation :
```
/home/user > ls -l
[sortie du ls -l...]
/home/user > cd Documents
/home/user/Documents > exit
```

#### Points clés à retenir :
1. Interface basique de type shell
2. Gère le changement de répertoire (`cd`) comme cas spécial
3. Capture et affiche à la fois la sortie standard et les erreurs
4. Utilise `subprocess.run()` avec `shell=True` (attention aux risques de sécurité si utilisé en production)
