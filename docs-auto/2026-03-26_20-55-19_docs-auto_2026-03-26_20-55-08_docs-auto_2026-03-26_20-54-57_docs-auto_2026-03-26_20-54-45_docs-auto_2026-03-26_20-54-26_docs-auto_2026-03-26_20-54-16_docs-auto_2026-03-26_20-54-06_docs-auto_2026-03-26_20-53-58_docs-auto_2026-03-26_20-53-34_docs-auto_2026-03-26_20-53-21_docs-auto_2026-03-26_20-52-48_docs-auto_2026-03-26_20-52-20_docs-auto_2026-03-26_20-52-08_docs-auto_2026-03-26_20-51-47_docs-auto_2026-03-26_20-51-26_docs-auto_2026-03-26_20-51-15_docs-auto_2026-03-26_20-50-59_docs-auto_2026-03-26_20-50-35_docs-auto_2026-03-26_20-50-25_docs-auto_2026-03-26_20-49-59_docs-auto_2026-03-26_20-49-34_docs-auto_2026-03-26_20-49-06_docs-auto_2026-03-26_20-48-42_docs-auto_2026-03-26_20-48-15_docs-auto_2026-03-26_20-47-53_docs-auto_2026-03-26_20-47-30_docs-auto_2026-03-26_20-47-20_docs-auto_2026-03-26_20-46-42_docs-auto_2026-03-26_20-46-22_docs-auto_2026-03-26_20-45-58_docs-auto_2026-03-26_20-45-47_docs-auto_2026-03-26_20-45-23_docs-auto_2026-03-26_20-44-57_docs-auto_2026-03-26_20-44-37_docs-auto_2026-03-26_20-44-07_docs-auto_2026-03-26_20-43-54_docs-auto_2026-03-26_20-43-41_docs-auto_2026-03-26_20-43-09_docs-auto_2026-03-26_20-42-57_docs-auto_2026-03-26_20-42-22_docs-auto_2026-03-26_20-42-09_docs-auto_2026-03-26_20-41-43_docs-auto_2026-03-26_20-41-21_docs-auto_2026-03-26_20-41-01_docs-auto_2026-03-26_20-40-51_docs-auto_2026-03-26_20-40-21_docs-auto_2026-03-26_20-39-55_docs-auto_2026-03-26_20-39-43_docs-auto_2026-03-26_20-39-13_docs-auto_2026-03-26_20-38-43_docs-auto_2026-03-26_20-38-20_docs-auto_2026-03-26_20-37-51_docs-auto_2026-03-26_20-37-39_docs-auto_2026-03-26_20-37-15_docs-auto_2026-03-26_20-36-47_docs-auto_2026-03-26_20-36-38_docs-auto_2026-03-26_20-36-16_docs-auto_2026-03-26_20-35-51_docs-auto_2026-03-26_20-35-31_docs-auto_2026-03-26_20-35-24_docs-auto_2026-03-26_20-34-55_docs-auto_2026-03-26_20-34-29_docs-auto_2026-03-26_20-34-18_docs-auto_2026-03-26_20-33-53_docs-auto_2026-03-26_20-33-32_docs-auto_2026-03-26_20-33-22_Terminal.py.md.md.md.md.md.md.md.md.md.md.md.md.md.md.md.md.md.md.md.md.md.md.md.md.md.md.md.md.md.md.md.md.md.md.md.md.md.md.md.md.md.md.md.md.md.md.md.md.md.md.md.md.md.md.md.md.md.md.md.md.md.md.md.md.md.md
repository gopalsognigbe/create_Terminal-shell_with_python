Analyse concise du fichier `Terminal.py` :

### Rôle principal
Le fichier implémente un terminal interactif minimaliste permettant :
- La navigation dans le système de fichiers
- L'exécution de commandes système

### Logique clé
1. **Cycle REPL** (Read-Eval-Print Loop) :
   - Affiche le répertoire courant comme prompt
   - Lit les commandes en continu

2. **Routing des commandes** :
   - Commandes internes (ex: `cd`) traitées directement
   - Autres commandes déléguées au shell sous-jacent

### Fonctions critiques
1. `change_directory(path)` :
   - Gère la navigation entre répertoires
   - Capture les erreurs de chemins invalides

2. `execute_command(command)` :
   - Point central d'exécution
   - Utilise `subprocess.run()` avec shell=True (danger)

### Exemple type
```bash
/home/user > ls -la  # Commande système standard
/var > cd /tmp       # Commande interne
```

### Problème majeur
**Vulnérabilité critique** :
- Injection de commandes possible via :
  - `shell=True` sans protection
  - Aucune sanitisation des entrées
- À proscrire en production

### Recommandation immédiate
Remplacer par :
```python
subprocess.run(command.split(), shell=False)
```
et implémenter une validation stricte des entrées.
