### Analyse synthétique du fichier `Terminal.py` :

#### **Rôle**  
Shell minimaliste en Python implémentant une boucle REPL pour exécuter des commandes système basiques (navigation, pipelines).

#### **Logique clé**  
1. Boucle infinie affichant une invite (`cwd >`)  
2. Détection des commandes internes (ex: `cd`) vs externes  
3. Exécution via `subprocess.run` (avec risques de sécurité)

#### **Fonctions principales**  
- `change_directory()` : Gère la navigation entre répertoires  
- `execute_command()` : Lance les commandes externes avec `shell=True` (critique)

#### **Exemple type**  
```bash
/tmp > ls -l | grep ".txt"  # Pipeline classique
```

#### **Avertissements**  
1. **Vulnérabilité** : Injection de commandes possible via `shell=True`  
2. **Limite** : Absence de gestion d'erreurs avancée ou de fonctionnalités shells modernes  

**Conclusion** : POC éducatif, non sécurisé pour un usage réel. À utiliser uniquement comme référence d'apprentissage.
