### Analyse concise du fichier `Terminal.py` :

**Rôle du fichier**  
Implémente un shell minimaliste avec des fonctionnalités de base :
- Navigation dans le système de fichiers
- Exécution de commandes système
- Gestion rudimentaire de pipelines

**Logique principale**  
Boucle REPL (Read-Eval-Print Loop) qui :
1. Affiche un prompt personnalisé
2. Lit l'entrée utilisateur
3. Traite les commandes spéciales (`cd`, `exit`)
4. Exécute les autres commandes via `subprocess`

**Fonctions clés**  
1. `change_directory(path)`  
   - Gère le changement de répertoire avec vérification d'existence
   - Utilise `os.chdir()` avec gestion d'erreur

2. `execute_command(command)`  
   - Exécution des commandes via `subprocess.run()` 
   - Support basique des pipelines avec `shell=True`
   - *Problème de sécurité* : `shell=True` expose aux injections

**Exemple d'utilisation**  
```bash
$ cd /tmp && ls -l | grep .txt  # Pipeline + changement de dossier
$ exit  # Quitter
```

**Points critiques**  
- Sécurité : L'implémentation actuelle est vulnérable aux attaques par injection
- Performances : L'utilisation de `shell=True` est coûteuse
- Suggestions :  
  - Remplacer par `subprocess.run(command.split())`  
  - Implémenter une whitelist de commandes autorisées
