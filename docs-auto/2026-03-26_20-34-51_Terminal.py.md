Analyse du fichier Terminal.py :

**Rôle du fichier** :
C'est un terminal simplifié qui permet d'exécuter des commandes système dans un environnement interactif, avec une gestion basique du changement de répertoire (cd) et de sortie (exit).

**Logique principale** :
1. Boucle infinie qui attend des commandes utilisateur
2. Traitement spécial pour la commande `cd` (changement de répertoire)
3. Traitement spécial pour la commande `exit` (sortie du terminal)
4. Exécution des autres commandes via subprocess.run()

**Fonctions importantes** :
- `os.chdir()` : changement de répertoire
- `subprocess.run()` : exécution des commandes système
- Gestion basique des erreurs pour `cd`

**Exemple d'utilisation** :
```
/home/user > ls
file1.txt file2.txt
/home/user > cd Documents
/home/user/Documents > exit
```
(Le terminal affiche le répertoire courant avant chaque commande et exécute les commandes normales du shell)

**Points à noter** :
- Il y a une erreur de syntaxe dans le code (`stderrb` au lieu de `stderr`)
- Le message "ex jjitnb pour sortir" semble être une erreur ou un texte parasite
- Le code utilise `shell=True` ce qui peut présenter des risques de sécurité
