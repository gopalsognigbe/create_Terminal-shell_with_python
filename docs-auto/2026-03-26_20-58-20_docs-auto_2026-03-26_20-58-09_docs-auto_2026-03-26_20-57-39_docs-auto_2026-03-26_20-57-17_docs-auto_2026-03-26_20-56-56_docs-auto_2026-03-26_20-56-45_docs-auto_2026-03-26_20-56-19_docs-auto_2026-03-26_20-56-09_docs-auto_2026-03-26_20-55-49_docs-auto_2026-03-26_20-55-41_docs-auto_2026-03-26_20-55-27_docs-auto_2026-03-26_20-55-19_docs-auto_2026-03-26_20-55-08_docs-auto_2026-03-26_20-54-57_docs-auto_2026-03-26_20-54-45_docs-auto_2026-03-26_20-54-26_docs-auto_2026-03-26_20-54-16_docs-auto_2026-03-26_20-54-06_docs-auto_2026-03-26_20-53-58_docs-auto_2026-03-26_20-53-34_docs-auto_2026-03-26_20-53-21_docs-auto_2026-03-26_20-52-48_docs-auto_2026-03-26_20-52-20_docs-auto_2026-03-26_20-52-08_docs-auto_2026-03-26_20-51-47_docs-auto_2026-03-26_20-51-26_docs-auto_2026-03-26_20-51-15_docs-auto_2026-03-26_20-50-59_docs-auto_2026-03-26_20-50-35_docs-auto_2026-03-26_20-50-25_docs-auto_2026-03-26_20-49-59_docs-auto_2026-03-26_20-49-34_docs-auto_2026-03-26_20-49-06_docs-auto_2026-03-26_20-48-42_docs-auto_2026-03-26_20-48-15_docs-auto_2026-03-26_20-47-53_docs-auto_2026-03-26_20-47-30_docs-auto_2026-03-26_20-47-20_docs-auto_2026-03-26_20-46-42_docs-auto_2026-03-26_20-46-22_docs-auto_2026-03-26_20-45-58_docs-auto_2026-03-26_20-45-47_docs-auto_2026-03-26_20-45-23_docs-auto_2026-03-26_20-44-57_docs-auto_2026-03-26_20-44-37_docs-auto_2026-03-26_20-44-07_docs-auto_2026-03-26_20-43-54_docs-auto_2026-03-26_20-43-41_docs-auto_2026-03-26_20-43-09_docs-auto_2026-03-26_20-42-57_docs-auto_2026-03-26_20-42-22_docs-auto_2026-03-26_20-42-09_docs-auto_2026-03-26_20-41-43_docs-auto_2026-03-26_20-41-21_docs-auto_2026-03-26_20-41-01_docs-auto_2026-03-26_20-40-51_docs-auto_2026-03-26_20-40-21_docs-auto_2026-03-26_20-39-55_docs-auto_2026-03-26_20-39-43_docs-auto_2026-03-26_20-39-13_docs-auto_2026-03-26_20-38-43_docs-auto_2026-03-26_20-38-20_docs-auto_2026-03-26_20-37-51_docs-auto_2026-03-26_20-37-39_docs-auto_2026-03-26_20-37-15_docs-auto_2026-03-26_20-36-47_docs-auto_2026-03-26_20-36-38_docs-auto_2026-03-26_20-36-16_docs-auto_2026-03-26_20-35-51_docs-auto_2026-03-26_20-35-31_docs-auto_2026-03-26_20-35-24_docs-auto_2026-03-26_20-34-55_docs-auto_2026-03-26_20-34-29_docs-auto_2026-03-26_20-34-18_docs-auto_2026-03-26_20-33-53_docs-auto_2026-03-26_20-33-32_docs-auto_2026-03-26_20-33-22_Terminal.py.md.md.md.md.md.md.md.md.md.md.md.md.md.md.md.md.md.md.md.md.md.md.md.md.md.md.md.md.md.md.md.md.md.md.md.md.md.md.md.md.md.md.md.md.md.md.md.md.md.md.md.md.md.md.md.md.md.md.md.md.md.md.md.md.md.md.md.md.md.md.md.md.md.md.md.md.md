### Analyse synthétique du fichier `Terminal.py` :

#### **Rôle du fichier**  
Implémente un terminal interactif simplifié pour naviguer dans le système de fichiers et exécuter des commandes système.

#### **Logique principale**  
1. Boucle REPL (Read-Eval-Print Loop) pour interpréter/exécuter les commandes en continu.  
2. Deux types de commandes :  
   - `cd` géré en interne pour changer de répertoire  
   - Autres commandes transmises au shell système via `subprocess`.

#### **Fonctions clés**  
1. **`change_directory(path)`**  
   - Gère la navigation entre répertoires  
   - Met à jour dynamiquement le prompt  
   - Inclut des vérifications de sécurité (existence du chemin)

2. **`execute_command(cmd)`**  
   - Point critique : utilise `subprocess.run` avec `shell=True` (risque d'injection)  
   - Devrait être remplacé par `subprocess.run(shlex.split(cmd), shell=False)`

#### **Exemple type**  
```bash
/user/docs > ls -la    # Liste les fichiers
/user/docs > cd /tmp   # Navigation
/tmp > pwd             # Commande système
```

#### **Alerte sécurité**  
La présence de `shell=True` constitue une vulnérabilité critique nécessitant une refactorisation immédiate comme indiqué ci-dessus.
