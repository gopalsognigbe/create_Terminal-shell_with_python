### Analyse concise du fichier `Terminal.py` :

#### **Rôle du fichier**  
Le fichier `Terminal.py` est un shell minimaliste qui fournit une interface REPL (Read-Eval-Print Loop) pour exécuter des commandes système simples, telles que la navigation entre répertoires (`cd`) et des pipelines basiques.

#### **Logique principale**  
Le programme repose sur une boucle REPL qui :  
1. **Affiche l'invite** : Montre le répertoire courant (`pwd`) pour guider l'utilisateur.  
2. **Traite la commande** : Identifie si la commande est interne (par exemple, `cd`) ou externe (par exemple, `ls`).  
3. **Exécute et affiche** : Exécute la commande et retourne le résultat à l'utilisateur.

#### **Fonctions importantes**  
1. **`change_directory(path)`**  
   - Permet de changer de répertoire en vérifiant que le chemin spécifié existe.  
2. **`execute_command(command)`**  
   - Exécute des commandes externes en utilisant `subprocess.run(shell=True)`, ce qui introduit un risque de sécurité dû à l'injection de commandes.

#### **Exemple d'utilisation**  
```bash
$ cd /tmp && ls -l | grep ".log"  # Utilisation d'un pipeline basique
```

#### **Avertissements**  
- **Sécurité** : L'utilisation de `shell=True` dans `subprocess.run` expose à des risques d'injection de commandes, ce qui rend ce script inadapté à un usage en production.  
- **Limitations** : Le shell est très rudimentaire et ne supporte pas des fonctionnalités avancées ou complexes.  

En résumé : Ce fichier est un Proof of Concept (POC) éducatif destiné à illustrer le fonctionnement d'un shell basique. Il ne doit pas être utilisé en production en raison de ses vulnérabilités et limitations.
