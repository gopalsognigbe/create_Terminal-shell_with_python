### Analyse concise du fichier `Terminal.py` :

#### **Rôle**  
Shell minimaliste offrant une interface REPL pour exécuter des commandes système (navigation, pipelines basiques).

#### **Logique centrale**  
Boucle REPL simple :  
1. Affiche l'invite (`pwd`)  
2. Traite la commande (interne/externe)  
3. Renvoie le résultat  

#### **Fonctions clés**  
1. **`change_directory(path)`**  
   - Gère la navigation entre répertoires avec vérifications basiques.  
2. **`execute_command(command)`**  
   - Exécute les commandes via `subprocess.run(shell=True)` (risque de sécurité).  

#### **Exemple type**  
```bash
$ cd /tmp && ls -l | grep ".log"  # Pipeline basique
```

#### **Avertissements**  
- **Vulnérabilités** : Injection de commandes possible via `shell=True`.  
- **Limitations** : Fonctionnalités très basiques (non adapté à un usage avancé).  

En résumé : POC éducatif, non sécurisé pour la production.
